# 测试Prompt：专家圆桌会议模式

## 基础测试

```
请按照 supser/dlt-expert-roundtable/SKILL.md 为大乐透第26033期召开专家圆桌会议。

上期开奖：26032期 前区 03 04 19 26 32 + 后区 01 12

要求：
1. 严格执行5个阶段的完整流程
2. 每位专家必须基于各自的SKILL.md方法论
3. 至少产生6组有实质内容的质疑与回应
4. 确保三个阵营有明显的策略差异
5. 最终输出15+5、9+3、5+2三种推荐
```

## 简化测试（快速验证）

```
请按照 supser/dlt-expert-roundtable/SKILL.md 为大乐透第26033期召开简化版圆桌会议。

执行阶段：
- 第一阶段：9位专家独立分析
- 第二阶段：3组关键质疑
- 第四阶段：投票统计
- 第五阶段：最终推荐

要求输出最终的15+5、9+3、5+2推荐。
```

## 聚焦测试（针对特定争议）

```
请按照 supser/dlt-expert-roundtable/SKILL.md 召开圆桌会议，重点讨论以下争议：

1. 08号问题：
   - 保守派观点：多维度热码，必选
   - 激进派观点：过热风险，应规避
   
2. 07号问题：
   - 激进派观点：遗漏31期，强烈回补
   - 保守派观点：过冷风险，不宜选择

请让专家们充分辩论这两个号码，最后通过投票决定是否纳入推荐。
```

## 验证要点

执行后检查以下内容：

### ✅ 结构完整性
- [ ] 包含所有必需章节
- [ ] 9位专家都有发言
- [ ] 有完整的投票统计表
- [ ] 有最终推荐结果

### ✅ 多样性验证
- [ ] 9位专家的初步推荐不完全相同
- [ ] 至少有3个不同的号码组合
- [ ] 三个阵营有明显差异

### ✅ 质量验证
- [ ] 论据引用了具体数据
- [ ] 质疑有理有据
- [ ] 回应正面回答了质疑
- [ ] 投票理由清晰

### ✅ 逻辑验证
- [ ] 专家观点符合其方法论
- [ ] 数据引用准确
- [ ] 推理过程合理
- [ ] 最终推荐符合共识原则

## 预期问题与解决

### 问题1：专家观点过于一致
**解决**：在prompt中强调
```
请确保专家之间有真实的分歧。保守派必须强调热码，激进派必须强调冷码，不要让所有人都选择相同的号码。
```

### 问题2：质疑流于形式
**解决**：要求具体的质疑
```
质疑必须针对具体的号码和具体的数据，不能只是泛泛而谈。例如："你选择08号，但它在XX维度的数据是XX，这与你的方法论矛盾"。
```

### 问题3：投票结果不合理
**解决**：检查投票逻辑
```
每位专家的投票必须与其初步推荐和阵营立场一致。如果专家在第一阶段推荐了某个号码，在投票阶段应该继续支持。
```
