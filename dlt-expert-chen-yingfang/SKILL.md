详见 `references/replay.md`。预测最新一期前，先运行 `python d:\ZyProject\supser\scripts\update_dlt_replays.py --skill dlt-expert-chen-yingfang` 刷新逐期记录和一二等中奖记录；`replay.md` 必须视为按当前使用日期生成的动态快照，而不是固定历史样本。

## 使用场景

- 用户要按陈樱芳的方法快速写一篇大乐透方案。
- 用户要复盘一种维度少但模板极稳的结构派方法。
- 用户想研究跨度、连号、重号这三类前区结构约束如何快速转成号码池。

## 必读输入

- 上一期大乐透开奖号码。
- 最近10期跨度分布。
- 最近10期连号出现次数。
- 最近10期重号出现次数。
- 最近10期后区大小结构。
- 当日刷新后的 `references/replay.md`，用它承接最新逐期记录、推荐位历史和一二等中奖记录。

## 专家核心思路

### 核心定位

- 陈樱芳是轻量结构派代表，她的核心不是多维复杂建模，而是用最少模块快速判断前区结构是否成形。
- 她的文章长期稳定围绕 `跨度 + 连号 + 重号 + 后区` 四件套展开，风格非常克制。
- 从往期文本看，她更像是在做“快速结构筛查”，而不是做全维度压缩系统。

### 分析框架

- 固定顺序是：`开奖回顾 -> 最近10期号码分析 -> 跨度推荐 -> 连号分析 -> 重号分析 -> 后区推荐 -> 15+5 -> 9+3 -> 5+2`。
- 前区真正的主轴只有三个：跨度定张力，连号定局部连接，重号定与上期的承接程度。
- 后区单独作为收尾模块，不和前区争夺篇幅。

### 核心分析逻辑

- 跨度是第一层，因为它决定前区 5 个号整体拉得多开，是陈樱芳最核心的总量约束。
- 连号判断负责看号码池内部会不会出现相邻粘连，这是对跨度的局部修正。
- 重号判断负责回答“本期是否还会从上期号码里留痕”，属于与上期关系的延续层。
- 后区则通过最近10期大小形态判断本期是继续大大、小小，还是回摆成大小搭配。

### 出号心理和策略

- 陈樱芳的出号心理很直接，她更愿意先把“整体拉开程度”和“是否要带上期痕迹”讲清楚，再给号码。
- 她不靠大量统计表堆结论，而是靠几个结构判断快速形成号码池。
- 她的策略不是极限杀号，而是先让跨度、连号、重号三件事不要互相打架，然后再把后区两码补上。
- 真正的信心点通常来自跨度判断明确、连号和重号方向也明确，这时她的轻量模板会明显更锋利。

### 命中一二等时的强化特征

- 从 25145、25126、24130 这几篇二等奖样本看，陈樱芳中奖时没有改用更复杂的方法，仍然是“跨度 + 连号 + 重号 + 后区”四件套。
- 她真正的命中点在于把三个前区结构信号写得更明确：跨度通常给到较清晰的单点附近，连号与重号也更敢直接判断“开”还是“不开”。
- 中奖样本里后区形态也更直接，比如明确押 `小小数` 或 `大大数` 两码，说明她的执行优势在于把简单结构判断贯彻到底，而不是增加维度。
- 与普通期相比，差异主要是判断更果断、组合更集中；研究陈樱芳时要重点看跨度、连号、重号三者是否形成互相支持，而不是单独看某一项。

## 分析步骤

1. 写开奖回顾时，先明确上一期有没有重号、有没有连号、跨度是多少，这三件事会直接成为后文判断的触发器。
2. 列出最近10期号码分析时，要把跨度、连号、重号的出现频率作为背景表，而不是只摆原始奖号。
3. 做跨度推荐时，先看最近10期跨度偏大还是偏小，再看上期跨度是否偏离常态，最后落成“本期跨度大致开在某个值附近”的判断。
4. 做连号分析时，要先判断最近10期连号出现是否频繁，再决定本期是继续带连号、弱连号，还是直接不防连号。
5. 做重号分析时，要先看最近10期重号出现次数和上期是否开重，再决定本期是继续防重，还是转向去重。
6. 做后区推荐时，只围绕最近10期后区大小结构或大小偏态给出两码主推，不扩写复杂尾数或和值逻辑。
7. 把跨度、连号、重号三层前区判断叠加成 `15+5大复式推荐`，再用同一逻辑压成 `9+3小复式推荐` 和 `5+2单挑一注推荐`。

### 步骤中的死扣细节

- 跨度必须是第一判断层，因为它决定前区整体张力，连号和重号都要服从跨度。
- 连号和重号不能只报统计次数，必须明确给方向判断，比如“继续防连号”或“重号降温”。
- 本 skill 里的“重号”默认指“本期前区与上一期前区重复的号码”；最近10期重号次数，也是按逐期与其上一期对比的口径统计，不把“与最近10期任意一期重复”混进来。
- 后区虽然轻，但必须和最近10期后区大小形态挂上钩，不能直接凭感觉给两码。
- 后区大小分界统一按 `01-06` 为小、`07-12` 为大，和 `d:\ZyProject\supser\GLOSSARY.md` 保持一致。
- `5+2` 必须从前面的复式池里收下来，不能另外拼一注和正文结构脱节的号码。

## 取舍规则

- 只使用跨度、连号、重号和后区形态，不补写龙头凤尾、区间、杀号等维度。
- 连号和重号可以判断“开”或“不开”，也可以点名具体候选，但必须先写方向。
- 后区主推通常是两码，不做过多扩展解释。
- 如果跨度结论与连号/重号冲突，优先保留跨度，再用连号和重号做细修。
- 预测最新一期前必须先刷新 `references/replay.md`，不能直接沿用旧日期快照。

## 可迁移经验

### 核心经验总结

- 用跨度做第一层总量约束，再用连号和重号做局部修正，是陈樱芳最重要的执行节奏。
- 轻量方法并不等于随意，关键是每个模块都要负责一个非常明确的问题。
- 她的方法特别适合快速初筛，因为判断链短，结论落地快。
- 后区保持极简，会让整篇文章一直把火力集中在前区结构上。

### 复制时的执行习惯

- 每次写作都先问自己：本期跨度要不要放大，连号还值不值得防，重号还值不值得留。
- 如果这三个问题没有连成一条线，就不要急着给小复式和单注。
- 陈樱芳的方法最怕被误写成“模块少所以随便写”，复制时反而要更死扣顺序和方向判断。

## 局限与禁忌

- 维度偏窄，对区间、龙头凤尾、和值、尾数等覆盖不足。
- 如果跨度判断失误，后续号码池容易整体偏移。
- 不要硬加冷热、历史同期或杀号模块，否则会破坏陈樱芳方法的简洁性。
- 这套方法适合做轻量结构约束，不适合当作全维度模型使用。
