# Supser Data Schema

## Purpose

This document describes the shared snapshot and support structures generated by the `supser`
scripts. It is intended to make expert skill references auditable and reduce ambiguity when a
skill cites fields from `dlt-support.json`, `pl3-support.json`, or per-run snapshot files.

## Main Files

- `supser/data/dlt-support.json`: Latest cached DLT support dataset.
- `supser/data/pl3-support.json`: Latest cached PL3 support dataset.
- `supser/predictions/_contexts/dlt/<issue>/snapshot-support.json`: DLT snapshot frozen for a
  specific run target.
- `supser/predictions/_contexts/pl3/<issue>/snapshot-support.json`: PL3 snapshot frozen for a
  specific run target.

## DLT Snapshot Top-Level Fields

- `asOfDate`: Snapshot date in `YYYY-MM-DD`.
- `lottoType`: `dlt`.
- `predictionIssueNo`: Target issue for prediction.
- `snapshotMode`: `latest` or `historical`.
- `sourceCurrentNextIssueNo`: The source cache's current next issue number.
- `historyCount`: Count of usable historical draws in the snapshot.
- `latestIssueNo`: Most recent draw included in the snapshot.
- `latestOpenTime`: Open time of `latestIssueNo`.
- `nextIssueNo`: Same as `predictionIssueNo`.
- `history`: Full normalized DLT history slice used by this run.
- `recent10`: First 10 entries of `history`.
- `recent10Summary`: Aggregated 10-issue summary.
- `recent10ZoneSupport`: Three-zone support details for the latest 10 issues.
- `jiangChuanPatternSupport`: Pattern-follow support used by the DLT Jiang Chuan expert.
- `sameIssueSupport`: Same-calendar-issue support used by Chen Qingfeng.
- `omissionSupport`: Omission-based support used by omission/cold-hot style experts.
- `transition460`: Long-window transition support used by Wan Miaoxian.
- `currentTransitionSnapshot`: Current transition view derived from `transition460`.

## DLT Normalized History Record

Each item in `history` / `recent10` contains:

- `issueNo`
- `openTime`
- `front`: Array of 5 ints.
- `back`: Array of 2 ints.
- `frontDisplay`
- `backDisplay`
- `frontSum`
- `frontSpan`
- `frontOddEven`: `{ odd, even, ratio }`
- `frontBigSmall`: `{ small, big, ratio }`
- `frontRoute012`: `{ r0, r1, r2, ratio }`
- `zone3`: `{ values, ratio }`
- `zone5`: `{ values, ratio }`
- `head`
- `tail`
- `frontConsecutivePairs`
- `frontRepeatWithPrevious`
- `backSum`
- `backSpan`
- `backOddEven`: `{ odd, even, ratio }`
- `backBigSmall`: `{ small, big, ratio }`
- `backRoute012`: `{ r0, r1, r2, ratio }`
- `frontTailOddEven`
- `frontTailRoute012`
- `frontTailBigSmall`

## DLT Support Sections

### `recent10Summary`

10-issue aggregate counters, including:

- `frontOddEven`
- `frontBigSmall`
- `zone3`
- `zone5`
- `heads`
- `tails`
- `backOddEven`
- `backBigSmall`

### `recent10ZoneSupport`

Per-zone latest-10 support:

- `zone1`, `zone2`, `zone3`
- Each zone includes `range`, `issueHits`, `hitCountTotal`, `appeared`, `missing`, `omissions`,
  `topMissing`

### `sameIssueSupport`

Same-calendar-issue support for DLT:

- `recordCount`
- `issueNos`
- `zone5Totals`
- `frontTailOddEven`
- `frontTailRoute012`
- `frontTailBigSmall`
- `backTailCounts`

### `omissionSupport`

- `front`
- `back`
- `frontTopMissing`
- `backTopMissing`
- `backGroups`
  - `hot`
  - `warm`
  - `cold`

### `transition460`

Long-window transition map keyed by current-condition patterns. Used for "condition transition"
style experts.

### `currentTransitionSnapshot`

Current draw's transition lookup result. Contains the latest front/back condition keys and the
mapped historical transition result.

### `jiangChuanPatternSupport`

Pattern-follow support for the DLT Jiang Chuan expert:

- `referenceIssueNo`
- `frontSizeFollow`
  - `currentRatio`
  - `occurrenceCount`
  - `recentNextDrawCount`
  - `recentNextDraws`
  - `hotNumbers`
- `frontOddEvenFollow`
  - `currentRatio`
  - `occurrenceCount`
  - `recentNextDrawCount`
  - `recentNextDraws`
  - `hotNumbers`
- `backPairFollow`
  - `currentPair`
  - `occurrenceCount`
  - `recentNextDrawCount`
  - `recentNextDraws`
  - `hotNumbers`

Important: `recentNextDraws` is a structured sample chain and should be treated as valid local
evidence by prompt logic. It is not merely a summary field.

## PL3 Snapshot Top-Level Fields

- `asOfDate`
- `lottoType`: `pl3`
- `predictionIssueNo`
- `snapshotMode`
- `sourceCurrentNextIssueNo`
- `historyCount`
- `latestIssueNo`
- `latestOpenTime`
- `nextIssueNo`
- `history`
- `recent20`
- `recent18`
- `positionSupport`

## PL3 Normalized History Record

Each PL3 history item contains:

- `issueNo`
- `openTime`
- `digits`: Array of 3 ints.
- `display`
- `shape`
- `sum`
- `span`
- `oddEven`
- `bigSmall`
- `route012`
- `positions`
  - `hundreds`
  - `tens`
  - `units`

## PL3 `positionSupport`

- `recent20`
  - `hundreds`
  - `tens`
  - `units`
- `recent18`
  - `hundreds`
  - `tens`
  - `units`

Each position block is generated from normalized records and is the canonical structured support
source for the PL3 Jiang Chuan expert.

## Notes

- Shared field existence can also be checked in `supser/scripts/validate_skill_suite.py`.
- If a skill depends on a field not documented here, the right fix is usually to document the
  field or adjust the skill to cite the existing structured source more precisely.
- Missing structured detail should not be assumed from `snapshot-support.md` alone; callers are
  expected to read `snapshot-support.json` when the prompt allows it.
